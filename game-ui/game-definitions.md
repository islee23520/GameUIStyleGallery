# Game Definition Gallery

Primary role: studio-indexed game-identity database.

## Repository Boundary

This gallery is a text-centric evidence record. It defines what each catalogued game IS at the level of identity, platform, engine, presentation, scenario, and core mechanics. It does not copy assets, source hierarchies, or implementation files from any game, and it does not treat one studio's engine or UI as a universal reference for other studios.

## Reusable Method

Each entry is a Game Definition Record synthesized from independent reputable sources. Capture facts as `[observed]` when a source states them directly, `[inferred]` when the record synthesizes them, and `[uncertain: reason]` when sources conflict or are scarce. Prefer two independent sources per fact, name the world and the primary conflict in scenario lore rather than plot beats, and describe UI elements by their on-screen role rather than by file or widget names. When the cataloguer's request conflicts with the sources, surface the correction in a `Corrections` block rather than silently adopting either side.

## Minimum Record

| Field | Required content |
| --- | --- |
| Identity | Canonical title, transliteration, developer, publisher, first release date, expansions |
| Platform | Engine, launch platforms, perspective, resolution and color depth at launch |
| Genre | Primary genre and one or two secondary descriptors |
| Graphics style | Two or three visual-language tags plus a one-sentence description with `[observed]` or `[inferred]` |
| UI elements | HUD regions, inventory and character sheets, navigation surfaces, input conventions, and any distinctive UI vocabulary |
| Scenario lore | The named world, primary antagonist or tension, and protagonist role; mark inferred and uncertain claims |
| Core mechanics | Three to five bullets naming the systems that define play |
| Multiplayer | Subscription and server model, PvP or party rules where applicable |
| Notable facts | Two to three bullets for contested claims, milestones, or common misattributions |
| Corrections | Common misattributions for this title, each with wrong vs correct vs correction source |
| Sources | Two or more independent reputable URLs per entry, each with a one-line relevance note |

## Cataloguer Corrections

This gallery was requested with several titles whose common English attributions do not survive fact-checking. The corrections below are surfaced here in addition to the per-record `Corrections` blocks, because the cataloguer's premise was the input and the correction is a first-class record value.

- `천년` requested as **Taihan Information Technology / AiON, 1997**. Correct attribution per sources: **Actoz Soft, 2000**. The "Taihan/AiON 1997" claim was not corroborated by any reviewed reputable source; GameMeca's contemporaneous 2000 preview and 2016 closure report both name Actoz and 2000.
- `조선협객전` requested as **"Legend of Chun-Hyang," by Joymax, 1996**. Correct attribution per sources: **developer and publisher Tomis Information & Telecom Corp., Korean commercial launch July 1998; archived English title "The Hero of Chosun."** "Legend of Chun-Hyang" is a mistranslation: 춘향 (Chun-hyang) is the heroine of the unrelated tale *Chunhyangjeon*, not the title of 조선협객전. Joymax's archived corporate history does not list this game.
- `어둠의 전설` requested with **1999** date. Correct attribution per sources: **Korean commercial service began January 1998** (3 January 1998 per Korean reporting); the 1999 date is the launch of the separately designed US adaptation *Dark Ages: Online RPG* by Nexon Inc., not the Korean original.
- `바람의 나라` developer credit: the cataloguer's premise suggested "Kim Hyun-tae at Nexon." The reviewed Nexon archive, Korean/English reference entries, and the developer's own interview identify the key early developer as **Jake Song / Song Jae-kyung (송재경)**, who later moved to NCsoft and led *Lineage*. The "Kim Hyun-tae" attribution was not corroborated.
- Origin Systems founders: the cataloguer's premise listed "Richard Garriott + his brother + Chuck Bueche." The complete founder set per sources is **Richard Garriott, Robert Garriott, Owen Garriott (father), and Chuck Bueche**.
- `Ultima Online` "first MMORPG" claim: contested. *Neverwinter Nights* (AOL, 1991) and *Meridian 59* (1996) predate it. The defensible distinction is that *UO* helped define and popularize the modern commercial graphical MMORPG.

#### Diablo (1996)

- Identity: *Diablo* (Korean 디아블로). Developer `[observed]` Blizzard North. Publisher `[observed]` Blizzard Entertainment. First release `[observed]` 1996-12-31 (North America, Windows). Expansion `[observed]` *Diablo: Hellfire* 1997-11-24, developed and published by Sierra On-Line (not Blizzard North).
- Platform: Engine `[inferred]` custom proprietary 2D isometric action-RPG engine (no public canonical engine product name documented). Launch platforms `[observed]` Windows 1996-12-31, Classic Mac OS 1997 (MacSoft port), PlayStation 1998-01 (Climax Studios port). Perspective `[observed]` fixed isometric 2D using sprite characters, monsters, props, and effects. Resolution `[observed]` 640×480, 256 colors on the Windows release.
- Genre: `[observed]` Action RPG / hack-and-slash dungeon crawler.
- Graphics style: `[inferred]` Dark gothic pre-rendered-look sprite art over 2D isometric dungeon tilemaps; cathedral, catacomb, cave, and Hell environments; deliberately low-light and oppressive.
- UI elements: `[observed]` Red Life globe and blue Mana globe at the lower corners; belt for immediately usable potions and scrolls; inventory and equipment paper-doll; character-stat sheet; spellbook; quest log; gold inventory; automap; left/right mouse attack-or-spell assignments with a spell-selection interface.
- Scenario lore: `[observed]` Set in Sanctuary, chiefly beneath the town of Tristram. The player is a Warrior, Rogue, or Sorcerer investigating corruption beneath Tristram's cathedral, where Archbishop Lazarus released the Prime Evil Diablo from imprisonment. The hero descends to defeat Diablo and attempts to contain his essence with the Soulstone; later series canon identifies the Warrior's fate as the beginning of the Dark Wanderer's story.
- Core mechanics: `[observed]` randomized dungeon layouts, monster populations, item drops, and a variable subset of quests; `[observed]` three fixed classes (Warrior, Rogue, Sorcerer) with distinct attribute profiles, weapons, magic access, and advancement; `[observed]` real-time click-to-move combat with melee/ranged/spell options and corpse/loot interaction; `[observed]` equipment and consumable loot management occupying limited inventory space; `[observed]` on death the character returns to town and must retrieve its dropped equipment (not the later dedicated Hardcore/permadeath mode).
- Multiplayer: `[observed]` up to 4 players via Battle.net plus contemporary local/network options; cooperative dungeon clearing and player-versus-player killing coexist in shared-game rules rather than a separately balanced competitive mode.
- Notable facts: `[observed]` Blizzard North's early design was turn-based and roguelike-influenced before changing to real-time play; `[observed]` *Diablo* was among the first games supported by Blizzard's original Battle.net service, which launched alongside it; `[observed]` *Hellfire* is an officially licensed expansion but not a Blizzard North-developed first-party expansion.
- Corrections: "Diablo is a 1997 game" → `[observed]` first North American Windows commercial release was 1996-12-31 (the frequent 1997 date reflects subsequent regional/Mac releases and cataloging conventions) — <https://www.mobygames.com/game/489/diablo/>. "Blizzard Entertainment developed Diablo" → `[observed]` Blizzard North developed it; Blizzard Entertainment published it — <https://en.wikipedia.org/wiki/Blizzard_North>.
- Sources: <https://www.mobygames.com/game/489/diablo/> (release/platform/credit catalog) · <https://en.wikipedia.org/wiki/Diablo_(video_game)> (overview of development, release, gameplay, story, ports, Battle.net) · <https://www.gdcvault.com/play/1023455/Classic-Game-Postmortem-Diablo> (GDC postmortem by David Brevik) · <https://web.archive.org/web/19970211135457/http://www.blizzard.com/diablo/> (archived Blizzard product page) · <https://en.wikipedia.org/wiki/Blizzard_North> (studio history).

#### Diablo II (2000) + Lord of Destruction (2001)

- Identity: *Diablo II* (Korean 디아블로 II); expansion *Diablo II: Lord of Destruction* (Korean 디아블로 II: 파괴의 군주). Developer `[observed]` Blizzard North. Publisher `[observed]` Blizzard Entertainment. First release `[observed]` 2000-06-29 (Windows, North America). Expansion `[observed]` *Lord of Destruction* 2001-06-27 (Windows and Mac OS), adding Act V, the Assassin and Druid classes, new items, runes/runewords, and a higher supported display resolution.
- Platform: Engine `[inferred]` proprietary 2D isometric engine evolved from *Diablo*; no formal public engine name consistently documented. Launch platforms `[observed]` Windows 2000-06-29, Classic Mac OS 2000-07; *Lord of Destruction* shipped for Windows and Mac OS 2001-06-27. Perspective `[observed]` isometric 2D with pre-rendered 3D-look sprites and backgrounds. Resolution `[observed]` base *Diablo II* 640×480, 256-color; `[observed]` *Lord of Destruction* adds 800×600 mode.
- Genre: `[observed]` Action RPG / hack-and-slash dungeon crawler.
- Graphics style: `[inferred]` Dark fantasy/horror pre-rendered sprite art on isometric maps, with five visually distinct regional acts: wilderness and monastery, desert, jungle, hellish ruins, and Mount Arreat.
- UI elements: `[observed]` red Life globe, blue Mana globe, stamina display; four-column belt with accessible potions, scrolls, and belt rows determined by belt type; equipment paper-doll; backpack inventory; stash; Horadric Cube; character sheet; quest log; automap; left/right mouse skill slots and keyboard skill hotkeys; mercenary panel; party interface, chat, and trade windows in multiplayer.
- Scenario lore: `[observed]` In Sanctuary, the hero pursues the corrupted Dark Wanderer (the former *Diablo* hero carrying Diablo's essence) eastward from the ruins of Tristram. One of five base-game heroes travels through Khanduras, Lut Gholein, Kurast, and Hell to stop Diablo from restoring himself. *Lord of Destruction* continues immediately afterward: Baal reaches Mount Arreat and seeks to corrupt the Worldstone; the hero enters the Worldstone Keep, kills Baal, and Tyrael destroys the damaged Worldstone.
- Core mechanics: `[observed]` five base classes (Amazon, Barbarian, Necromancer, Paladin, Sorceress) each with three skill trees; *Lord of Destruction* adds Druid and Assassin; `[observed]` randomized maps, monster groups, loot, dungeon layouts, and some quests support repeated runs; `[observed]` buildcraft centers on attribute allocation, skill points, equipment affixes, sockets, gems, runes, runewords, sets, uniques, and crafted items; `[observed]` acts use town hubs, waypoints, town portals, NPC quests, hireable mercenaries, and boss progression; `[observed]` difficulty progression includes Normal, Nightmare, and Hell; Hardcore characters have permanent death.
- Multiplayer: `[observed]` Battle.net supports up to 8 players; TCP/IP and local-network play supported in original-era releases; cooperative questing, trading, parties, and voluntary hostility/PvP coexist in the same game sessions; town areas are protected.
- Notable facts: `[observed]` *Lord of Destruction* is a first-party Blizzard North expansion, unlike Sierra's externally developed *Diablo: Hellfire*; `[observed]` the expansion's 800×600 mode, Act V, Druid, Assassin, runes, runewords, jewels, charms, and elite items substantially reshaped the enduring version of *Diablo II*; `[observed]` the original game and expansion were later remastered together as *Diablo II: Resurrected*, which is not the original 2000/2001 release's technical presentation.
- Corrections: "Diablo II was developed by Blizzard Entertainment in Irvine" → `[observed]` Blizzard North developed, Blizzard Entertainment published — <https://en.wikipedia.org/wiki/Blizzard_North>. "800×600 was the base Diablo II launch resolution" → `[observed]` base shipped at 640×480; 800×600 was added by *Lord of Destruction* — <https://www.mobygames.com/game/607/diablo-ii/>. "The Assassin is one of the five original classes" → `[observed]` Assassin and Druid were introduced by *Lord of Destruction* — <https://en.wikipedia.org/wiki/Diablo_II:_Lord_of_Destruction>.
- Sources: <https://www.mobygames.com/game/607/diablo-ii/> (MobyGames record) · <https://www.mobygames.com/game/6254/diablo-ii-lord-of-destruction/> (expansion record) · <https://en.wikipedia.org/wiki/Diablo_II> (release, Blizzard North development, systems, plot, multiplayer) · <https://en.wikipedia.org/wiki/Diablo_II:_Lord_of_Destruction> (expansion release, content, technical changes) · <https://web.archive.org/web/20000815053938/http://www.blizzard.com:80/diablo2/> (archived Blizzard product page) · <https://diablo2.blizzard.com/en-us/> (current official franchise page).

#### Diablo III (2012) + Reaper of Souls (2014)

- Identity: *Diablo III* (Korean 디아블로 III); expansion *Diablo III: Reaper of Souls* (Korean 디아블로 III: 영혼을 거두는 자). Developer `[observed]` Blizzard Entertainment. Publisher `[observed]` Blizzard Entertainment. First release `[observed]` 2012-05-15 (Windows and Mac OS X, worldwide regional launch). Expansion `[observed]` *Reaper of Souls* 2014-03-25 (Windows and macOS); console content arrived in *Diablo III: Ultimate Evil Edition* 2014-08-19.
- Platform: Engine `[observed]` Blizzard proprietary/in-house *Diablo III* engine with custom physics and destructible environments; no separately marketed canonical engine title documented. Launch platforms `[observed]` Windows and Mac OS X 2012-05-15; later PlayStation 3 and Xbox 360 2013-09-03; PlayStation 4 and Xbox One via *Ultimate Evil Edition* 2014-08-19; Nintendo Switch *Eternal Collection* 2018-11-02. Perspective `[observed]` 3D isometric action-RPG view with fixed/controlled elevated camera. Resolution `[observed]` resolution-scalable; Blizzard's launch minimum display requirement was 1024×768; `[uncertain: a definitive official launch color-depth specification was not located]`.
- Genre: `[observed]` Action RPG / hack-and-slash dungeon crawler.
- Graphics style: `[inferred]` stylized 3D fantasy environments and characters with painterly textures, strong silhouette readability, physics debris, and a more colorful outdoor palette than the first two games.
- UI elements: `[observed]` red Life globe and a class-specific resource globe (Fury, Hatred/Discipline, Spirit, Mana, Arcane Power, or Wrath) at the bottom; six active skill slots (left/right mouse buttons plus four action-bar hotkeys) with a separate potion slot; upper-right minimap and objective/quest tracker; inventory, equipment paper-doll, detailed character panel, skill/rune interface, achievements, crafting/artisan interfaces, stash, follower panel; health globes drop from enemies, replacing the previous games' belt-centered potion-bar combat loop.
- Scenario lore: `[observed]` Set 20 years after *Diablo II* in Sanctuary, the player is a Nephalem arriving in New Tristram after a falling star strikes the old cathedral. The hero aids Leah and Deckard Cain, pursues the return of the Prime Evils, and ultimately destroys Diablo, reborn as the Prime Evil through the Black Soulstone. In *Reaper of Souls*, the fallen angel of Wisdom Malthael steals the Black Soulstone and uses death magic to reap humanity in Westmarch; the Nephalem defeats him before he can exterminate Sanctuary's human population.
- Core mechanics: `[observed]` five base classes (Barbarian, Demon Hunter, Monk, Witch Doctor, Wizard); *Reaper of Souls* adds Crusader; `[observed]` skills unlock by level and can be modified by non-item skill runes, with active loadouts changeable outside combat constraints; `[observed]` randomized map segments, enemy packs, elite affixes, loot, rifts, and difficulty systems support replayability; `[observed]` loot includes normal, magic, rare, legendary, and set items; *Reaper of Souls* added Loot 2.0-era improvements, enchanting, transmogrification, Adventure Mode, Nephalem Rifts, and the Mystic; `[observed]` Hardcore characters become permanently unplayable after death.
- Multiplayer: `[observed]` original Windows/Mac release required a Battle.net connection even for solo play and supports up to 4-player online cooperative parties; console editions support local cooperative play; no finished PvP Arena at launch; patch 1.0.7 added Brawling (Feb 2013), a scoreless free-for-all for 2–4 players.
- Notable facts: `[observed]` always-online requirement on PC/Mac caused launch-server failures commonly associated with the "Error 37" message; `[observed]` Blizzard operated both gold and real-money auction houses after launch; both were closed on 2014-03-18, shortly before *Reaper of Souls*; `[observed]` *Reaper of Souls* raised the level cap from 60 to 70 and added the Crusader, Act V/Westmarch, Adventure Mode, and Rifts.
- Corrections: "Diablo III launched with PvP Arena combat" → `[observed]` Arena was delayed then cancelled; Brawling arrived in patch 1.0.7 in 2013 — <https://en.wikipedia.org/wiki/Diablo_III>. "The real-money auction house was removed in 2013" → `[observed]` closure announced September 2013, both auction houses remained available until 2014-03-18 — <https://news.blizzard.com/en-us/diablo3/10974978/diablo-iii-auction-house-shutting-down-march-18>. "All versions of Diablo III require a permanent online connection" → `[observed]` true for the original PC/Mac release; console editions permit local/offline play — <https://en.wikipedia.org/wiki/Diablo_III>.
- Sources: <https://en.wikipedia.org/wiki/Diablo_III> (release, engine, systems, auction-house, PvP, platform, story) · <https://en.wikipedia.org/wiki/Diablo_III:_Reaper_of_Souls> (expansion release date, platform rollout, scenario, class, features) · <https://us.diablo3.blizzard.com/en-us/game/guide/gameplay/> (official gameplay guide) · <https://web.archive.org/web/20120318042714/http://us.battle.net/d3/en/blog/4612389/> (archived 2012-05-15 launch announcement) · <https://news.blizzard.com/en-us/diablo3/10974978/diablo-iii-auction-house-shutting-down-march-18> (auction-house shutdown) · <https://www.polygon.com/2015/8/4/9097497/diablo-3-sales-30-million-units> (commercial history context).

#### StarCraft (1998) + Brood War (1998)

- Identity: *StarCraft* (Korean 스타크래프트); expansion *StarCraft: Brood War* (Korean 스타크래프트: 브루드 워). Developer `[observed]` Blizzard Entertainment; *Brood War* co-developed by Blizzard Entertainment and Saffire. Publisher `[observed]` Blizzard Entertainment for Windows/Mac; Nintendo published the later *StarCraft 64* port. First release `[observed]` 1998-03-31 (Windows, North America). Expansion `[observed]` *Brood War* 1998-12-18 (Windows, US); Mac OS release 1999-06.
- Platform: Engine `[observed]` custom Blizzard RTS engine. Development began using *Warcraft II: Tides of Darkness* engine code as a base, but Blizzard substantially redesigned it for *StarCraft* (overhaul by programmer Bob Fitch to support burrowing, cloaking, and other features). Launch platforms `[observed]` Windows 1998-03-31, Classic Mac OS 1999-03-12; *Brood War* Windows 1998-12-18, Mac OS 1999-06; *StarCraft 64* Nintendo 64 2000-06-13 (North America). Perspective `[observed]` 2D isometric projection using pre-rendered sprites and backgrounds. Resolution `[observed]` 640×480, 256 colors on the original PC release.
- Genre: `[observed]` Real-time strategy (RTS).
- Graphics style: `[observed]` pre-rendered sprite-and-background art with deliberately distinct faction styling: industrial gray Terrans, organic/insectoid Zerg, gold/blue high-technology Protoss.
- UI elements: `[observed]` bottom-left minimap with terrain, unit, alert, and visibility information; top resource display for minerals, vespene gas, and supply; selected-unit portrait; shields/hit points/status panel; unit command buttons; construction options; research/upgrades; production queues; control groups; selection boxes; rally points; unit orders; alliances; chat; multiplayer-lobby controls.
- Scenario lore: `[observed]` *StarCraft* takes place in the Koprulu Sector, where exiled human Terrans, the insectoid Zerg, and advanced psionic Protoss collide. The player commands all three factions over three campaigns: Confederate marshal Jim Raynor becomes a rebel, Sarah Kerrigan is captured and infested by the Zerg, and Protoss executor Tassadar allies with Zeratul and sacrifices himself to destroy the Zerg Overmind on Aiur. *Brood War* continues after the Overmind's death: Kerrigan, now the Queen of Blades, defeats the United Earth Directorate and rival powers to secure Zerg dominance in the sector.
- Core mechanics: `[observed]` three asymmetric factions (Terran, Zerg, Protoss) with different economies, construction rules, units, technology trees, and tactical strengths; `[observed]` harvest minerals and vespene gas, manage supply, construct bases, research upgrades, and produce armies in real time; `[observed]` fog of war and terrain/chokepoints make scouting, information control, positioning, and timing strategically important; `[observed]` single-player campaigns use scripted mission objectives and cinematics; multiplayer uses custom and melee maps; `[observed]` *Brood War* adds new campaigns, terrain tilesets, music, technologies, and seven units, including Terran Medics/Valkyries, Zerg Lurkers/Devourers, and Protoss Corsairs/Dark Archons.
- Multiplayer: `[observed]` up to 8 players via Battle.net plus contemporary direct/LAN connection options; modes include free-for-all, teams, and scenario/custom maps; standard melee victory is generally achieved by eliminating opponents' forces/buildings under the match's rules; *StarCraft 64* has local split-screen cooperative support but lacks online multiplayer.
- Notable facts: `[observed]` the 1996 E3 build was criticized as "Warcraft in space," leading Blizzard to overhaul the project and strengthen faction differentiation; `[observed]` Blizzard first based the project on *Warcraft II* code, then redesigned the engine rather than shipping a simple reskin; `[observed]` *StarCraft* became a defining South Korean esports title, with professional play, television broadcasts, sponsorships, and dedicated leagues.
- Corrections: "StarCraft simply uses the unmodified Warcraft II engine" → `[observed]` it began from *Tides of Darkness* engine code, but Blizzard redesigned the engine to implement burrowing, cloaking, and other features — <https://web.archive.org/web/20010210182346/http://www.blizzard.com/blizz-anniversary/bob.shtml>. "Brood War launched in November 1998" → `[observed]` official US release was 1998-12-18; November dates appear through preview/listing/regional catalog ambiguity — <https://en.wikipedia.org/wiki/StarCraft:_Brood_War>. "The original PC game has only two symmetric sides like Warcraft II" → `[observed]` *StarCraft* has three intentionally asymmetric factions, a central design response to the early E3 criticism — <https://www.gamespot.com/articles/starcraft-preview/1100-2563222/>.
- Sources: <https://en.wikipedia.org/wiki/StarCraft_(video_game)> (release, platforms, development, engine lineage, gameplay, story, multiplayer, South Korean esports) · <https://en.wikipedia.org/wiki/StarCraft:_Brood_War> (expansion release, co-development, campaign continuation, units, platform history) · <https://web.archive.org/web/19990210065137/http://209.67.136.168/PRESS/980331sc.htm> (archived Blizzard press release) · <https://web.archive.org/web/20080402134120/http://www.blizzard.com/us/press/10-years-starcraft.html> (archived Blizzard 10-year retrospective) · <https://web.archive.org/web/20010210182346/http://www.blizzard.com/blizz-anniversary/bob.shtml> (archived developer account of engine redesign) · <https://www.gamespot.com/articles/starcraft-preview/1100-2563222/> (contemporary GameSpot preview).

#### Ultima mainline series

- Identity: *Ultima* mainline series (Korean 울티마); precursor *Akalabeth: World of Doom* (1979 hobbyist development / 1980 commercial publication). Developer `[observed]` Richard Garriott developed *Akalabeth*, *Ultima I*, and *Ultima II* before Origin existed; Origin Systems, Inc. developed/published the core series beginning with *Ultima III: Exodus*; EA acquired Origin in September 1992 and published *Ultima IX*. Publisher per entry: *Akalabeth* California Pacific Computer Company (commercial 1980); *Ultima I* California Pacific initially, Origin reissued later; *Ultima II* Sierra On-Line initially, Origin reissued later; *Ultima III–VIII* Origin Systems; *Ultima IX* Electronic Arts. Expansions: `[observed]` *Ultima VII: The Black Gate* — *Forge of Virtue* (1992); *Ultima VII Part Two: Serpent Isle* (1993) is a direct sequel/subsequent main-series chapter rather than an expansion, with *The Silver Seed* as its expansion; *Ultima VIII: Pagan* — *The Lost Vale* was planned but cancelled/unreleased; *Ultima IX* — no released expansion.

#### Ultima Online (1997)

- Identity: *Ultima Online* (Korean 울티마 온라인). Developer `[observed]` Origin Systems, Inc. (1997–2004); subsequent stewardship moved through Electronic Arts, Mythic Entertainment, and Broadsword. - Core mechanics: `[observed]` create a persistent character using a skill-based progression system rather than fixed classes; `[observed]` explore a shared world, fight monsters, gather resources, craft goods, trade, and participate in a player economy; `[observed]` build/own houses and organize guilds, with substantial player-created social structures; `[observed]` open-world PvP, criminality, murder-count/reputation systems, and later separated PvP/PvE facets; `[observed]` live events and ongoing expansion-driven world additions.
- Multiplayer: `[observed]` persistent subscription MMORPG run on separate server "shards"; early published monthly fee $9.95. At launch, PvP was broadly available; *Renaissance* later separated Felucca's full-risk PvP ruleset from Trammel's substantially safer PvE-oriented facet.
- Notable facts: `[observed]` launched 1997-09-24, nearly two years before *EverQuest*'s March 1999 launch; `[observed]` one of the earliest graphical MMORPGs and a major commercial breakthrough for Internet-only games, reaching 100,000 subscribers by December 1998; `[observed]` should not be labeled the unqualified "first MMORPG" — *Neverwinter Nights* and *Meridian 59* predate it; its defensible distinction is helping define and popularize the modern commercial graphical MMORPG.
- Corrections: "Ultima Online was the first MMORPG" → `[observed]` contested; *Neverwinter Nights* (1991) and *Meridian 59* (1996) predate it — <https://en.wikipedia.org/wiki/Meridian_59>, <https://en.wikipedia.org/wiki/Neverwinter_Nights_(1991_video_game)>. "UO is a full 3D MMORPG" → `[observed]` Classic Client is 2D sprite-and-tile; 3D presentation arrived later via *Third Dawn* and *Kingdom Reborn* — <https://en.wikipedia.org/wiki/Ultima_Online>.

## Opinionated Guidance

- Index first by studio, then by title. A studio's engine, art direction, and UI vocabulary recur across its catalogue; the studio block records what is shared before each game record narrows to one title.
- Keep genre, platform, engine, graphics style, UI, scenario, and mechanics as separate fields even when they overlap; a genre label is not a graphics style and an engine is not a UI system.
- Mark `[observed]` for direct source statements and `[inferred]` for synthesis. Treat `[uncertain: reason]` as a first-class record value, not a gap to fill by guessing.
- When the cataloguer's premise conflicts with the sources, surface the correction in both the per-record `Corrections` block and the top-level `Cataloguer Corrections` section. The conflict is a first-class record value, not editorial noise.
- Cite at least two independent reputable sources per entry and attach a one-line relevance note per URL. Korean-language sources are valid and preferred for Korean titles.
- For titles with scarce sourcing (천년, 조선협객전), prefer a single primary archived source over a confident secondary paraphrase, and explicitly mark `[uncertain: source scarcity]` rather than imputing facts.

## Platform-Specific Guidance

Record the launch engine, render perspective, resolution, and color depth as implementation evidence. Note Battle.net, Battle.net 2.0, NCsoft's subscription model, Nexon's DOOMVAS architecture (where contested, mark contested), or studio-specific server and subscription models as multiplayer implementation facts. Do not promote any one studio's online infrastructure as universal architecture. For Korean PC-bang-era titles, record the Windows version, the PC-bang distribution model, and the Korean/international service split as separate implementation facts rather than collapsing them into a single "launch" date.

## Unsupported Absolutes

- A genre label does not establish a graphics style or a UI system.
- A studio's first commercial success does not prove it invented a genre; flag firsts carefully and distinguish "first Korean MMORPG" from "first commercially successful Korean MMORPG" and "longest-running Korean MMORPG."
- A single Korean-language source is acceptable evidence; absence of English sources is not absence of the game.
- Two studios using the same perspective (for example 2D isometric tile plus sprite) does not prove shared engine code.
- A localized title is not a translation of the original title (*Dark Ages* is not a translation of *어둠의 전설*; *Nexus: The Kingdom of the Winds* is the North American adaptation, not the original *바람의 나라*).
- A Korean and an international version of the same nominal franchise can have distinct settings, lore, and design (Korean *어둠의 전설* on Mysochia vs US *Dark Ages* on Temuair).
- "First MMORPG" claims are contested; *Neverwinter Nights* (1991) and *Meridian 59* (1996) predate *Ultima Online* (1997).
