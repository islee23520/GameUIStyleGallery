# Source Admissibility

Sources are admissible only inside a claim boundary.

## Source Families

- Academic and standards sources can support conceptual claims.
- Official standards and methodology documents can support accessibility, evaluation, and process claims.
- Official tool docs can support operational procedures.
- Vendor or practice docs can support implementation examples, not prove human design quality.
- Public design-system docs can support vocabulary and comparison, not brand imitation or affiliation.

## Required Source Boundary

For every cited source, record:

- what the source can support;
- what the source cannot prove;
- whether it is academic, standards, official tooling, or practice reference;
- whether the claim needs local rendered evidence or user evidence.

## Rejection Rules

Reject source use when:

- a blog post is used as proof of human design quality;
- a public design system is treated as permission to imitate a brand;
- a tool doc is used to prove accessibility or usability outcomes;
- a source is cited without a claim, warrant, or boundary.
